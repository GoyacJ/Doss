# Doss AI HR 开发方案（基于 PRD v1.0）

文档日期：2026-02-26  
关联文档：`docs/PRD.md`

## 1. 目标与范围

本方案用于落地 PRD 的 MVP 范围：
1. 四站采集（Boss/智联/58/拉勾）。
2. 自动简历初筛（T0/T1/精筛/加分/风险）。
3. 面试题库生成。
4. 面后评估。
5. 最终录用决策闭环。

非目标沿用 PRD：不做多账号协作、不做自动语音转写、不做 SaaS 化。

3. 面试题库、面后评估、最终决策的数据模型与命令缺失。
4. 页面上缺少“面试页/决策页”或等价扩展区块。

## 3. 核心设计原则

1. 优先复用现有组件，不新增视觉体系。
2. 优先扩展现有页面与 store，避免大规模重构。
3. 数据模型一次到位（screening/interview/decision），UI 分阶段接入。
4. 所有 AI 输出必须附证据与不确定性字段，满足可解释性要求。

## 4. 组件复用策略（重点）

### 4.1 复用优先级

1. 第一优先：现有 `Ui*` 组件直接复用。
2. 第二优先：在现有组件上做小幅增强（例如 `UiPanel` 的 header slot 扩展）。
3. 第三优先：仅在无法表达业务结构时新增组件。

### 4.2 页面复用落地

1. `JobsView.vue`
- 扩展“评分模板选择/微调”区块。
- 复用 `UiPanel + UiField + UiButton`。

2. `CandidatesView.vue`
- 增加“初筛结果卡片”“生成面试题库入口”。
- 复用 `UiPanel + UiInfoRow + UiBadge + UiButton`。

3. `SettingsView.vue`
- 增加“全局评分模板管理”“风险规则管理”。
- 复用现有表单组件，不新建设计系统。

4. 新增页面（最小化）
- `InterviewView.vue`（题库展示与面后提交）。
- `DecisionView.vue`（AI建议 + 最终录用/遗憾）。
- 仍使用 `UiPanel/UiTable/UiField/UiButton` 组合，不引入新 UI 库。

## 6. 关键业务流程落地

### 6.1 自动初筛

1. 简历入库（手动/文件/sidecar）完成后触发 `run_resume_screening`。
2. 返回结构包含：T0/T1/精筛/加分/风险、推荐、证据、核验建议。
3. 在候选人详情展示初筛卡片。

### 6.2 面试题库

1. 条件：候选人初筛为 `PASS` 或人工复核放行。
2. 生成内容：主问题、追问链、评分要点、红旗。
3. 支持再生成与手工编辑后保存。

### 6.3 面后评估 + 最终决策

1. 输入：转写文本 + 结构化评价（必填），录音路径（可选）。
2. 输出：`HIRE | HOLD | NO_HIRE` + 证据 + 核验点 + 不确定性。
3. HR 最终提交：`录用/遗憾 + 原因码 + 备注`，并记录 AI 偏差。

## 7. 分阶段实施（与 PRD 里程碑对齐）

1. M1（抓取能力）：lagou 适配 + 类型扩展 + 回归。
2. M2（初筛引擎）：模板/微调/风险规则 + 自动初筛 + 候选人展示。
3. M3（面试闭环）：题库生成 + 面后反馈 + AI评估。
4. M4（决策闭环）：最终决策、偏差统计、验收与发布。

## 8. 测试与验收策略

1. 单测扩展：

2. 集成验证：

3. 验收映射：严格对照 `docs/PRD.md` 的 11/12 章标准与场景。

## 9. 风险与控制

1. 拉勾页面结构波动：通过多 selector + normalize 容错 + 错误分类兜底。
2. AI 结果不稳定：强制输出 schema 校验，不足证据统一落 `HOLD`。
3. 页面复杂度上升：以“候选人页入口 + 独立面试/决策页”降低耦合。
4. 回归风险：保持现有流程 API 兼容，新增能力默认向后兼容。

## 10. 交付定义（DoD）

1. 四站抓取可用（含 lagou）。
2. 自动初筛输出完整五部分并可解释。
3. 面试题库可生成/编辑/保存。
4. 面后评估可产出建议与证据，不足证据必须 `HOLD`。
5. 最终决策可落库并追踪 AI 偏差。
6. 全链路审计可追溯。
