# PRD：Doss AI HR（单机桌面优先版）

文档版本：v1.0  
文档日期：2026-02-26

## 一、简要总结

Doss 是一个本地优先的 AI 招聘工作台，面向 HR/招聘负责人，提供“多站点采集 → 自动初筛 → 面试题库生成 → 面后评估 → 最终录用决策”的完整闭环。  
本 PRD 以当前桌面架构为基础，MVP 首发支持 Boss、智联、58、拉勾四站，默认自动触发初筛；评分体系采用“全局模板 + 职位微调”。

## 二、背景与现状

当前项目已具备：职位/候选人管理、三站采集（Boss/智联/58）、简历入库、AI 分析、阶段流转、审计与本地存储。  
当前缺口：拉勾适配、可配置化初筛规则（T0/T1/精筛/加分/风险）、因人面试题库、面后结构化评估、最终决策原因闭环。

## 三、产品目标与非目标

### 3.1 目标

1. 将 HR 从“简历筛选和面评整理”中解放，缩短筛选到面试的时间。  
2. 让初筛、面试、终审形成可回溯证据链。  
3. 通过统一评分框架提升不同岗位、不同面试官之间的一致性。

### 3.2 非目标（MVP 不做）

1. 多账号协作与权限体系。  
2. 自动语音转写（MVP 先支持粘贴转写文本）。  
3. 云端 SaaS 化和跨组织数据共享。  
4. Offer 审批流程与入职流程系统对接。

## 四、目标用户与核心场景

1. 用户角色：招聘 HR、招聘负责人。  
2. 核心场景：  
- 导入岗位并抓取候选人。  
- 自动初筛并给出可解释结论。  
- 对通过初筛候选人生成个性化题库并执行面试。  
- 面后输入转写和评价，得到 AI 录用建议。  
- HR 最终做“录用/遗憾”决策并沉淀原因。

## 五、端到端业务流程

1. 创建/选择岗位，配置岗位要求与评分模板微调。  
2. 发起四站抓取任务，候选人入库并做去重/归并。  
3. 简历入库完成后自动触发初筛。  
4. 输出初筛结果：通过/建议复核/不通过 + 证据 + 风险提示。  
5. HR 选择进入面试，系统生成候选人专属面试题库。  
6. 面试后提交：转写文本（必填）、结构化面试评价（必填）、录音文件（可选，MVP 可不解析）。  
7. AI 输出面后评估：建议录用/待定/不录用 + 证据 + 核验点 + 不确定性。  
8. HR 最终选择：录用/遗憾，并记录原因码与备注。  
9. 全流程事件进入审计与统计。

## 六、功能需求（FR）

### FR-1 多站点采集

1. 支持源站：Boss、智联、58、拉勾。  
2. 任务类型：职位抓取、候选人抓取、简历抓取。  
3. 支持合规模式与高级模式。  
4. 支持任务去重、失败重试、冲突归并、质量报告。  
5. 合规要求：仅账号登录后授权抓取，限速与错误分类。

### FR-2 自动简历初筛

1. 初筛构成：T0 硬性条件 + T1 核心评分 + 精细化筛选 + 加分项 + 风险提示。  
2. T1 支持动态增删维度与权重修改。  
3. 配置模型：全局模板 + 职位微调。  
4. 输出必须包含：分数、建议、证据片段、风险等级、核验建议。

### FR-3 面试题库生成

1. 对通过初筛或“建议复核通过”的候选人可生成题库。  
2. 题库结构：主问题、追问链、评分要点、红旗信号。  
3. 题目应结合岗位 JD、候选人简历亮点与风险点。  
4. 支持再次生成、手动增删改题目。

### FR-4 面后评估

1. 输入：转写文本（必填）、面试官结构化表单（必填）、录音文件（可选）。  
2. 输出：建议录用/待定/不录用。  
3. 输出附带：证据引用、需人工核验点、不确定性说明、置信度。  
4. 若证据不足，必须返回“待定+补充建议”。

### FR-5 最终决策与闭环

1. HR 最终动作：录用 / 遗憾。  
2. 必填原因码，支持备注。  
3. 记录 AI 建议与 HR 最终决策偏差，用于后续优化。  
4. 全流程保留审计日志。

## 七、评分体系与决策规则

### 7.1 T0 硬性条件（5 分制）

1. `T0 < 3`：不匹配，默认不通过。  
2. `3 ≤ T0 < 4`：建议复核。  
3. `T0 ≥ 4`：匹配，可进入综合打分。

### 7.2 T1 核心评分（默认维度与权重）

| 维度 | 权重 |
|---|---|
| 目标导向 | 30% |
| 团队协作 | 15% |
| 自驱力 | 15% |
| 反思迭代 | 10% |
| 开放性 | 8% |
| 抗压韧性 | 7% |
| 学习能力 | 10% |
| 价值观契合 | 5% |

评分规则：每维 1-5 分，换算到 0-100，按权重加权。  
支持新增维度，且总权重必须等于 100%。

### 7.3 精细化筛选（默认）

1. 学历匹配度。  
2. 工作年限匹配度。  
3. 行业跨度风险。  
4. 薪资合理性。  
默认综合为 0-100 分，用于拉开边界候选人差异。

### 7.4 加分项与风险提示

1. 加分项：高质量项目结果、关键证书、稀缺行业经验、关键岗位直接命中。  
2. 风险提示：频繁跳槽、关键信息缺失、薪资显著偏离、行业转换成本高、证据不足。  
3. 风险分级：低/中/高。高风险会触发扣分或直接建议“待定”。

### 7.5 初筛最终建议（MVP 默认）

1. `T0 < 3`：不通过。  
2. `T0 ≥ 3` 时计算综合分（T1、精筛、加分、风险扣减）。  
3. 输出区间：  
- `>=80` 且无高风险：通过初筛。  
- `65-79` 或中高风险：建议复核。  
- `<65`：不通过。

## 八、重要接口/类型/数据模型变更

### 8.1 公共类型（shared）

1. `SourceType` 增加 `lagou`。  
2. 新增 `ScreeningTemplate`、`ScreeningResult`、`InterviewKit`、`InterviewFeedback`、`InterviewEvaluation`、`HiringDecision` 类型。  
3. 新增推荐枚举：`PASS`、`REVIEW`、`REJECT` 与 `HIRE`、`HOLD`、`NO_HIRE`。

### 8.2 Sidecar 接口

1. 现有 `/v1/crawl/jobs|candidates|resume` 扩展支持 `lagou`。  
2. 错误码补充反爬/登录失效/权限不足分类。  
3. 统一返回抓取来源与证据快照。

### 8.3 Tauri Command（新增）

1. `get_screening_template` / `upsert_screening_template`  
2. `run_resume_screening` / `list_screening_results`  
3. `generate_interview_kit` / `save_interview_kit`  
4. `submit_interview_feedback` / `run_interview_evaluation`  
5. `finalize_hiring_decision`

### 8.4 SQLite 表（新增建议）

1. `screening_templates`、`screening_dimensions`、`job_screening_overrides`  
2. `screening_results`  
3. `interview_kits`  
4. `interview_feedback`  
5. `interview_evaluations`  
6. `hiring_decisions`

## 九、交互与页面需求

1. 职位页：增加“评分模板选择/微调”。  
2. 候选人页：新增“初筛结果卡片”和“生成面试题库”入口。  
3. 面试页（新增）：题库展示、追问链、评分要点、红旗记录、面后提交表单。  
4. 决策页（新增或候选人详情扩展）：AI 评估结论、证据、最终录用/遗憾操作与原因。  
5. 设置页：全局评分模板管理、风险规则管理。

## 十、非功能与合规要求

1. 性能：单候选人初筛结果返回 <= 5 秒（本地模型 fallback 可放宽）。  
2. 稳定性：抓取失败可重试，任务状态可追踪。  
3. 安全：手机号/邮箱继续加密存储，审计日志不可关闭。  
4. 合规：默认合规抓取模式，强调账号授权与平台条款遵循。  
5. 可解释性：所有 AI 结论必须附证据和不确定性说明。

## 十一、验收标准（Acceptance Criteria）

1. 四站抓取可用且能进入候选人入库闭环。  
2. 初筛输出包含完整五部分（T0/T1/精筛/加分/风险）与推荐结论。  
3. T1 维度可动态增删，权重可调整且校验总和=100%。  
4. 通过初筛候选人可生成个性化面试题库。  
5. 面后输入文本评价后可生成“建议录用/待定/不录用”。  
6. HR 可完成“录用/遗憾”决策并记录原因。  
7. 全链路可回溯（任务、分析、面评、决策、审计）。

## 十二、测试用例与场景

1. 抓取适配测试：Boss/智联/58/拉勾职位、候选人、简历路径全通过。  
2. 初筛规则测试：T0 临界值（2.9/3.0/3.9/4.0）与推荐结果正确。  
3. 权重测试：新增维度、删除维度、总和非 100 拒绝保存。  
4. 风险测试：高风险触发“建议复核/不通过”逻辑。  
5. 面试题库测试：同候选人重复生成稳定；不同候选人差异化明显。  
6. 面后评估测试：证据不足时必须给“待定+补充项”。  
7. 最终决策测试：录用/遗憾写入成功并可追溯 AI 建议偏差。  
8. 回归测试：现有职位池/候选人池/阶段流转功能不回退。  
9. 异常测试：站点登录失效、反爬封禁、网络抖动、AI 超时 fallback。  
10. 安全测试：敏感字段加密与日志脱敏持续有效。

## 十三、里程碑建议

1. M1：拉勾适配 + 类型/接口扩展 + 基础回归。  
2. M2：初筛引擎重构（模板、微调、风险规则、结果展示）。  
3. M3：面试题库生成与面后评估闭环。  
4. M4：最终决策、偏差统计、全链路验收与发布。

## 十四、明确假设与默认决策

1. 产品形态默认“单机桌面优先”。  
2. MVP 默认包含拉勾首发支持。  
3. T1 采用“全局模板 + 职位微调”。  
4. 简历入库后默认自动触发初筛。  
5. 面后输入在 MVP 以“粘贴转写文本 + 结构化评价”为主，录音文件仅可选存档。  
6. HR 最终决策始终保留人工覆盖权，AI 仅做建议。
